# ЗВІТ ПРО ВИКОНАННЯ ЛАБОРАТОРНОЇ РОБОТИ №2

**Тема:** Проєктування та розробка програм з використанням патернів проєктування
**Дисципліна:** Основи об'єктно-орієнтованого програмування
**Варіант:** Демонстрація та аналітичний аналіз роботи алгоритмів сортування

---

# 1. Структура проєкту та поділ на компоненти

Програма спроєктована за модульним принципом (**Loose Coupling**). Вся кодова база розділена на 5 взаємонезалежних файлів, що дозволяє команді розробників працювати над різними частинами системи без виникнення конфліктів у репозиторії.

| Файл            | Призначення                                               |
| --------------- | --------------------------------------------------------- |
| `config.py`     | Модуль глобальних налаштувань програми                    |
| `models.py`     | Рівень даних та кастомних структур                        |
| `algorithms.py` | Ядро обчислювальних алгоритмів та стратегій               |
| `core.py`       | Бізнес-логіка, диспетчеризація команд та фабрика об'єктів |
| `main.py`       | Графічний інтерфейс користувача на базі фреймворку Pygame |

---

# 2. Інструкція користувача (Як демонструвати програму)

Управління візуалізатором повністю перенесене на гарячі клавіші для уникнення проблем із рендерингом інтерфейсу операційної системи.

| Клавіша                 | Дія                                                                           |
| ----------------------- | ----------------------------------------------------------------------------- |
| `1`, `2`, `3`, `4`, `5` | Перемикання між алгоритмами Bubble, Insertion, Selection, Quick та Shell Sort |
| `SPACE`                 | Запуск покрокової візуалізації обраного сортування                            |
| `G`                     | Генерація нового випадкового масиву                                           |
| `U`                     | Скасування останньої дії (Undo) та повернення масиву до початкового стану     |

Поточний алгоритм відображається у верхньому лівому кутку вікна програми.

---

# 3. Візуальна мапа патернів: що та де показувати викладачу

Для отримання оцінки **«відмінно»** у проєкті задіяно **11 класичних патернів GoF** із трьох різних категорій.

| Патерн              | Клас / Метод                  | Демонстрація                                                                                               |
| ------------------- | ----------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Singleton**       | `SettingsManager`             | Зміна параметрів в одному місці миттєво впливає на всю програму, оскільки об'єкт існує в одному екземплярі |
| **Factory Method**  | `SortFactory.create_sorter()` | При перемиканні алгоритмів інтерфейс отримує об'єкти через фабрику                                         |
| **Strategy**        | `BaseSort` та його нащадки    | Алгоритм можна змінити без зміни логіки клієнтського коду                                                  |
| **Template Method** | `BaseSort.sort()`             | Визначає скелет алгоритму: `_before_sort() → _execute_sort() → _after_sort()`                              |
| **Adapter**         | `ShellSortAdapter`            | Адаптує стару процедурну функцію `legacy_shell_sort()` до нового інтерфейсу                                |
| **Decorator**       | `MetricsDecorator`            | Додає підрахунок операцій без зміни логіки сортування                                                      |
| **Facade**          | `SortFacade`                  | Надає простий інтерфейс `execute_sorting()` та `undo()`                                                    |
| **Command**         | `SortCommand`                 | Натискання пробілу створює окремий об'єкт-команду                                                          |
| **Memento**         | `ArrayMemento`                | Дозволяє повернути масив до початкового стану після натискання `U`                                         |
| **Iterator**        | `CustomArray.__iter__()`      | Pygame перебирає елементи структури даних через стандартний цикл                                           |
| **Observer**        | `StatsSubject`                | Завершення сортування автоматично оновлює статистику на екрані                                             |

---

# 4. Демонстрація персистентності (Збереження даних)

Після кожного успішного сортування програма автоматично записує результат у файл `history.json`.

Приклад вмісту файлу:

```json
[
    {
        "timestamp": "2026-06-01 20:45:12",
        "algorithm": "Quick",
        "array_size": 40,
        "operations": 114
    },
    {
        "timestamp": "2026-06-01 20:45:30",
        "algorithm": "Bubble",
        "array_size": 40,
        "operations": 384
    }
]
```

Це підтверджує виконання вимоги технічного завдання щодо збереження результатів роботи у файловій системі.

# 5. Результати верифікації (Юніт-тести)

Для перевірки коректності бізнес-логіки системи було реалізовано набір тестів у файлі `test_sort.py`.

Тести перевіряють:

* коректність роботи алгоритмів сортування;
* інваріанти структур даних;
* логіку відновлення стану (Undo / Memento);
* стабільність взаємодії між компонентами системи.

Результат запуску тестів:

```bash
andrew@MacBook-Air-Andrew lab2 % python3 -m unittest test_sort.py

....
----------------------------------------------------------------------
Ran 4 tests in 0.02s

OK
```

Усі тести пройдено успішно (`OK`), що підтверджує технічну стабільність, правильність реалізації алгоритмів та коректність обраної архітектури програмного забезпечення.
